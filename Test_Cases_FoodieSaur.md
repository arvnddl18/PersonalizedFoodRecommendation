# Test Cases - FoodieSaur Application

## Test Case Documentation

| Test Case ID | Test Scenario / Feature | Test Steps / Actions | Expected Result | Actual Result | Status | Remarks / Comments |
|--------------|-------------------------|---------------------|-----------------|---------------|--------|-------------------|
| TC-01 | AI Restaurant Recommendation | Log in → Chat Interface → Type "I'm hungry" or "Can you recommend something?" | The system should display personalized restaurant recommendations based on user preferences, location, and behavioral patterns | | | |
| TC-02 | Restaurant Details Viewing | Log in → Chat Interface → Click on a recommended restaurant → View Details | Restaurant details (name, address, rating, price level, photos, contact info) should be displayed in a card overlay | | | |
| TC-03 | Map Integration | Log in → Chat Interface → View recommended restaurants | Recommended restaurants should be displayed on Google Maps with custom markers and the user's current location | | | |
| TC-04 | Search Function | Log in → Chat Interface → Search for specific cuisine or restaurant type (e.g., "Italian restaurants" or "pizza places") | Restaurants matching the searched criteria should appear in the search results and on the map | | | |
| TC-05 | Responsiveness (Desktop/Mobile) | Access the website on desktop and mobile devices, navigate through different pages and features | The website should run smoothly without lags or delays during loading times, and all features should be accessible on both desktop and mobile | | | |
| TC-06 | Favorites Viewing | Log in → View Favorites Panel → Select a favorite restaurant → View Details | The favorite restaurant details should appear, and the restaurant should be marked with a heart icon on the map | | | |
| TC-07 | Page Navigation | Verify that users can navigate between pages (Home, Chat, Preferences, Account, etc.) smoothly | There shouldn't be any errors or "page not found" errors when navigating between pages | | | |
| TC-08 | Internet Connectivity | Test the website functionality with different internet connection speeds (slow, normal, fast) | No interruptions, no lags, and no delays in loading content or interacting with features | | | |
| TC-09 | Browser Compatibility | Test the website on different browsers (Chrome, Firefox, Safari, Edge) | No lags, and no delays - all features should work consistently across all browsers | | | |
| TC-10 | Performance (Speed / Stability) | Navigate through different pages and features, measure loading times | No lags, and no delays - each page should load within acceptable time limits (typically 2-3 seconds) | | | |
| TC-11 | Adding to Favorites | Log in → Chat Interface → View recommended restaurant → Click heart icon to add to favorites | The restaurant should be added to favorites, heart icon should fill, and it should appear in the Favorites Panel | | | |
| TC-12 | Account Log In | Fill up the email and password fields, click the log in button | The user should be able to access and view their account dashboard (Chat interface) | | | |
| TC-13 | Account Registration | Create account if not yet existing, fill up the necessary details (name, email, password, confirm password, reCAPTCHA). Redirects to the Log in page and log in with the account recently made | The user should be able to have their account created and log in with that account successfully | | | |
| TC-14 | Profile and Preferences | Log in → Account/Preferences → Fill necessary details (dietary preferences, food types, price range, location) → Save | Information should be displayed and saved, and recommendations should be personalized based on these preferences | | | |
| TC-15 | Favorites Interaction | Log in → Chat Interface → View recommended restaurant → Add to favorites → View Favorites Panel → Remove from favorites | The like/favorite should be added and removed successfully, and the favorites list should update accordingly | | |
